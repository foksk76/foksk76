# Привет, я foksk76

Я собираю практичные homelab и open-source проекты вокруг почтовой инфраструктуры, syslog, ELK и observability.

Смысл этих проектов — воспроизводимо прогонять подозрительную почтовую активность в Kerio Connect lab, проводить логи через Logstash и Elasticsearch до метрик и дашбордов в Grafana / Kibana, чтобы быстрее выявлять признаки компрометации, проводить разбор событий и снижать риск повторных атак.

## Основная линейка проектов: Kerio Connect Monitoring & Logging

Лабораторное семейство проектов для:

- запуска Kerio Connect в lab-окружении
- пересылки и парсинга syslog
- анонимизации реальных данных для безопасной публикации
- сборки дашбордов в Grafana и Kibana

## С чего начать

1. [kerio-connect](https://github.com/foksk76/kerio-connect) — запустить lab-окружение
2. [kerio-logstash-project](https://github.com/foksk76/kerio-logstash-project) — распарсить и нормализовать логи
3. [kerio-syslog-anonymizer](https://github.com/foksk76/kerio-syslog-anonymizer) — анонимизировать реальные данные для демо и публикации

## Поток проекта

```text
Kerio Connect lab -> Syslog -> Logstash -> Elasticsearch -> Grafana / Kibana
                                      -> анонимизированные примеры для публичных репозиториев и статей
```

## Для кого это

- Homelab-инженеры, которым нужен воспроизводимый стенд для email observability.
- Системные администраторы, которые изучают, как syslog превращается в удобные для поиска операционные данные.
- Security, DevSecOps и observability-практики, которым нужны безопасные примеры для демо или передачи проекта.
